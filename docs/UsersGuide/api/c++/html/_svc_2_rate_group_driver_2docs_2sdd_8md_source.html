2 # Svc::RateGroupDriver Component
3 
4 ## 1. Introduction
5 
6 The RateGroupDriver Component is used to take a single system tick and distribute it to multiple rate groups in a system.
7 It takes the input `Svc::Sched` port, then divides down the tick rate based on arguments to the constructor.
8 Typically, the output ports would be connected to the asynchronous inputs of an `ActiveRateGroup`.

38 #### 3.2 Functional Description
39 
40 The Svc::RateGroupDriver component has one input port that receives a system tick.
41 The constructor has an array of integer arguments that specifies the divisors for each output port.
42 
43  RateGroupDriverImpl(const char* compName, NATIVE_INT_TYPE dividers[], NATIVE_INT_TYPE numDividers);
44 
45 The input rate will for each output port will be divided down by the value in the `dividers[]` array corresponding to the output port number.
46 
47 The implementation will be ISR compliant by avoiding the following:
48 
49 * Floating point calculations
50 * Taking mutexes
51 * Calling libraries with unknown side effects
52 * Long implementation
53 
54 For instance,
55 
56 `SchedIn` Rate | `divider[0]` | `SchedOut[0]` | `divider[1]` | `SchedOut[1]` | `divider[2]` | `SchedOut[2]`
57 -------------- | ------------ | ------------- | ------------ | ------------- | ------------ | -------------
58 1Hz | 1 | 1Hz | 2 | 0.5Hz | 4 | 0.25Hz
