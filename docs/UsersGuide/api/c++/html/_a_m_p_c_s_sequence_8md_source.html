2 # Svc::AMPCSSequence Class
3 
4 `AMPCSSequence` is a derived class of `CmdSequencer::Sequence`.
5 It loads, validates, and runs AMPCS-compatible sequence files.
6 
7 <a name="File_Format"></a>
8 ## 1 File Format
9 
10 A sequence file consists of a 32-bit sequence header followed by
11 up to 32,768 bytes of record data.
12 
13 The record data consists of zero or more records.
14 Each record has the following format:
15 
16 Record Field | Size in Bytes | Description
17 ------------ | ------------- | -----------
18 Time Flag | 1 | Either 0x0 denoting absolute time or 0xFF denoting relative time
19 Time | 4 | The time in seconds
20 CMD Length | 2 | The size in bytes of the Command field
21 Command | < 1017 | A serialized command in F Prime format, with a 16-bit opcode
22 
23 ## 2 Implementation
24 
25 The `AMPCSSequence` class implements the virtual functions of
26 `CmdSequencer::Sequence` as follows.
27 
28 <a name="loadFile"></a>
29 ### 2.1 `loadFile(`*filename*`)`
30 
31 1. Read the stored CRC value from *filename*`.CRC32`.
32 
33 2. Ask the OS for the size of the sequence file, and use it to compute the size
34 of the command record data.
35 
36 3. Read the 32-bit sequence header.
37 
38 4. Read the command record data into a serial buffer *B*.
39 
40 5. Check that the format of the records is valid according to the binary format
41 specified in [File Format](#File_Format).
42 
43 6. Compute the CRC value of the binary file and check it against the stored value.
44 
45 7. Fill in `m_header` as follows:
46 
47  * `m_fileSize`: The size of the command record data, computed in item 2 above.
48 
49  * `m_numRecords`: Count the records when validating them.
50 
51  * `m_timeBase`: Use 0xFFFF (don't care).
52 
53  * `m_timeContext`: Use 0xFF (don't care).
54 
55 ### 2.2 `hasMoreRecords`
56 
57 Return `true` if and only if *B* has more data.
58 
59 ### 2.3 `nextRecord`
60 
61 Deserialize and return the next record stored in the serial buffer.
62 The deserialized record contains the following binary data in its
63 argument field:
64 
65 Field | Size in Bytes | Value
66 ----------------- | ------------- | -----------
67 Packet Descriptor | 4 | `Fw::ComPacket::FW_PACKET_COMMAND`
68 Opcode | 4 | The opcode stored in bytes 0-1 of the serial record, zero-extended to four bytes
69 Arguments | Variable | Bytes 2-end of the serial record
70 
71 ### 2.4 `reset`
72 
73 Reset *B* for deserialization.
74 
75 ### 2.5 `clear`
76 
77 Reset *B* for serialization.
