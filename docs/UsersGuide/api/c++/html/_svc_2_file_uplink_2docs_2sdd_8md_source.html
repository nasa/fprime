2 # Svc::FileUplink Component
3 
4 ## 1 Introduction
5 
6 `FileUplink` is an active ISF component.
7 It manages uplink of files to the spacecraft.
8 
9 ## 2 Requirements
10 
11 Requirement | Description | Rationale | Verification Method
12 ---- | ---- | ---- | ----
13 ISF-FU-001 | `FileUplink` shall receive file packets, assemble them into files, and store the files in the on-board non-volatile storage. | This requirement provides the capability to uplink files to the spacecraft. | Test
14 
15 ## 3 Design
16 
17 ### 3.1 Assumptions
18 
19 The design of `FileUplink` assumes the following:
20 
21 1. File uplink occurs by dividing files into packets
22 of type [`Fw::FilePacket`](../../../Fw/FilePacket/docs/sdd.html).
23 
24 2. In the nominal case of file uplink
25 
26  a. Files are received one at a time.
27 All packets of one file are received before receiving any
28 packets of the next file.
29 
30  b. Within a file, packets are received in order.

53 ### 3.4 State
54 
55 `FileUplink` maintains the following state:
56 
57 * <a name="receiveMode">*receiveMode*</a>:
58 One of START or DATA, recording the type of the next packet that
59 `FileUplink` expects to receive.
60 The initial value is START.
61 
62 * <a name="lastSequenceIndex">*lastSequenceIndex*</a>:
63 An integer recording the sequence index of the last packet received.
64 The initial value is zero.
65 
66 * <a name="writeFileDescriptor">*writeFileDescriptor*</a>:
67 The file descriptor of the file, if any, that is currently open
68 for writing.
69 
70 ### 3.5 The bufferSendIn Port
71 
72 `FileUplink` asynchronously receives buffers on
73 [`bufferSendIn`](#bufferSendIn).
74 Each buffer contains a file packet.
75 When `FileUplink` receives a buffer, it (a) determines the type
76 of the file packet in the buffer; (b) takes action as
77 specified in section below corresponding to the packet type; and (c)
78 invokes [`bufferSendOut`](#bufferSendOut)
79 to return the buffer for deallocation.
80 
81 #### 3.5.1 START Packets
82 
83 Upon receipt of a START packet, `FileUplink` does the following:
84 
85 1. If [*receiveMode*](#receiveMode) is not START,
86 then close the file at
87 [*writeFileDescriptor*](#writeFileDescriptor)
88 and issue an *InvalidReceiveMode* warning.
89 
90 2. Open the file for writing and set
91 [*writeFileDescriptor*](#writeFileDescriptor).
92 
93 3. If step 2 succeeded, then set
94 [*lastSequenceIndex*](#lastSequenceIndex)
95 to zero and go to DATA mode; otherwise issue a
96 *FileOpenError* warning and go to START mode.
97 
98 #### 3.5.2 DATA Packets
99 
100 Upon receipt of a DATA packet *P*, `FileUplink` does the following,
101 where *I* is the
102 [sequence index](../../../Fw/FilePacket/docs/sdd.html)
103 of *P*:
104 
105 1. If
106 [*receiveMode*](#receiveMode)
107 is not DATA, then issue an *InvalidReceiveMode* warning
108 and go to START mode.
109 
110 2. Otherwise
111 
112  a. If *I* is not equal to *lastSequenceIndex + 1*, then issue a
113 *PacketOutOfOrder*
114 warning reporting *lastSequenceIndex* and *I*.
115 
116  b. If the packet offset and size are in bounds for the current file, then
117 
118  1. Using *writeFileDescriptor*, write the file data in the
119 packet at offset specified in the packet.
120 
121  2. If there was an error writing the file, then issue a
122 *FileWriteError* warning.
123 
124  c. Otherwise issue a *PacketOutOfBounds* warning.
125 
126 #### 3.5.3 END Packets
127 
128 Upon receipt of an END packet *P*, `FileUplink` does the following:
129 
130 1. If [*receiveMode*](#receiveMode) is *DATA*,
131 then do the following, where *I* is the sequence index of *P*:
132 
133  a. If *I* is not equal to *lastSequenceIndex + 1*,
134 then issue a *PacketOutOfOrder* warning reporting
135 *lastSequenceIndex* and *I*.
136 
137  b. Use *writeFileDescriptor* to do the following:
138 
139  1. Use the method described in &sect; 4.1.2 of the
140 [CCSDS File Delivery Protocol (CFDP) Recommended Standard](http://public.ccsds.org/publications/archive/727x0b4.pdf)
141 to compute the checksum value for the file.
142 
143  2. Compare the value computed in the previous step against the
144 checksum value in the packet.
145 If the two values are different, then issue a *BadChecksum* warning.
146 
147  c. Close the file.
148 
149 2. Otherwise issue an *InvalidReceiveMode* warning.
150 
151 3. Set *lastSequenceIndex* to zero and go to START mode.
152 
153 #### 3.5.4 CANCEL Packets
154 
155 Upon receipt of a cancel packet *P*, `FileUplink` does the following:
156 
157 1. Set *lastSequenceIndex* to zero.
158 
159 2. If *receiveMode* is not START, then close the file at
160 *writeFileDescriptor*.
161 
162 3. Issue an *UplinkCanceled* event.
163 
164 4. Go to START mode.
