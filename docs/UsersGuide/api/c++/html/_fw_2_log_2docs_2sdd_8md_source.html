2 # Fw::Log / Fw::LogText Ports
3 
4 ## 1. Introduction
5 
6 The `Fw::Log` port is used to pass a serialized form of an ISF Event. It passes the ID, a time tag, the severity, and a buffer
7 containing the serialized arguments of the event.
8 
9 The `Fw::LogText` port is used to pass a printable text representation of an ISF Event. It passes the ID, a time tag,
10 the severity, and a string containing a description of the event.

22 The `Fw::Log` port has the following arguments:
23 
24 Argument | Description
25 ----- | -----------
26 id | Event identifier
27 timeTag | The system time the event happened
28 severity | The severity of the event.
29  | LOG_FATAL - A condition has been encountered that makes the software unable to continue
30  | LOG_WARNING_HI - A condition has been encountered that is a serious failure, but the software can continue
31  | LOG_WARNING_LO - A condition has been encountered that is a failure, but the software is largely unaffected
32  | LOG_COMMAND - An activity related to processing commands has happened
33  | LOG_ACTIVITY_HI - An important nominal event has happened
34  | LOG_ACTIVITY_LO - An unimportant nominal event has happened that is a subset of an important event
35  | LOG_DIAGNOSTIC - A detailed list of events in an activity that normally do not need to be seen; used for debugging
36 args | Serialized event arguments
