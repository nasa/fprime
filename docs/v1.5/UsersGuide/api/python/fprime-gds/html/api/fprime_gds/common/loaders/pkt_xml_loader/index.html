get_id_dict(self, path, ch_name_dict)[source]¶

Returns the python dictionary keyed by ids for the given path

This function will return the same dictionary originally computed for the given path or will construct new dictionaries if the path has never been passed to the get_id_dict or the get_name_dict functions.

Parameters

  • path (string) – Path to the xml pkt spec to convert to a python dict

  • ch_name_dict (dict()) – Channel dictionary with names as keys and ChTemplate objects as values.

Returns

The id dictionary associated with the given path

construct_dicts(self, path, ch_name_dict)[source]¶

Constructs and returns python dictionaries keyed on id and name

This function should not be called directly, instead, use get_id_dict(path) and get_name_dict(path)

Parameters

  • path (string) – Path to the packet definition xml file to parse into a dictionary.

  • ch_name_dict (dict()) – Channel dictionary with names as keys and ChTemplate objects as values.

Returns

(id_dict, name_dict). The keys should be the packets’ id and name fields respectively and the values should be PktTemplate objects.

Return type

A tuple with two packet dictionaries (type==dict())
